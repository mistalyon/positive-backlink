// Setup Wizard

Activate your PositiveBacklink platform

Four steps. Five minutes. After this, login works.

1

Paste your Supabase Anon Key

This key is public-safe (designed to be exposed in browser). Stored only in your browser localStorage, never sent to GitHub.

Where do I find this key?
  1. Open Supabase API Settings
  2. Scroll to "Project API keys"
  3. Find the row labeled anon public
  4. Click the copy icon next to it
  5. Paste below
2

Run database migrations

Create the users, sites, exchanges, notifications, and error tables.

  1. Open Supabase SQL Editor
  2. Open sql/schema.sql, copy ALL of it, paste in editor, click RUN
  3. Open sql/notifications-schema.sql, repeat
  4. Open sql/errors-schema.sql, repeat
3

Enable auth providers

Turn on Email + Google sign-in in Supabase.

  1. Open Supabase Auth Providers
  2. Find Email -> Enabled (already on by default). Set "Confirm email" to ON.
  3. Find Google -> Click to expand -> Toggle Enable. You will need Google Cloud OAuth credentials. Full guide. Skip Google for now if you do not have it - email login still works.

Then set the Site URL:

  1. Open URL Configuration
  2. Site URL: https://www.positivebacklink.com
  3. Redirect URLs - add each on its own line:
https://www.positivebacklink.com/dashboard https://www.positivebacklink.com/reset-password https://www.positivebacklink.com/verify-email http://localhost:3000/**
4

Test it!

Open /register, create an account, check your email, then /login and land in your dashboard.

If login fails: check the browser console (F12) for the error message and check that the anon key in Step 1 is correct.
